API Optimization

Why Your API Costs Are Higher Than They Should Be

The default SentiencePro config doesn't account for cost. The Server License includes a dedicated optimization report that fixes this.

What the Report Covers

Five specific areas where the default configuration wastes tokens without improving response quality. Each section includes recommended settings for your specific character count.

  • ✓
    Temperature Calibration
    Default temperature settings produce inconsistent responses. Lower for guards and merchants. Slightly higher for oracles. Character-specific tuning.
  • ✓
    Token Budget Per Character
    Max token limits set per NPC type. Short-response guards don't need the same budget as lore-heavy oracles. Reduces cost without reducing quality.
  • ✓
    Memory Retrieval Tuning
    How many memories get retrieved per query affects both cost and coherence. Optimal retrieval count differs by character type.
  • ✓
    Context Window Management
    Conversation history length settings. Too long wastes tokens. Too short breaks continuity. The report provides recommended lengths per NPC.
  • ✓
    Model Selection Guide
    Which NPCs benefit from gpt-4o-mini vs. Claude Haiku vs. larger models. Most servers are using models that are more expensive than necessary.
